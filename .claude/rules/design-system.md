# Design System

`DESIGN.md` is the source of truth for UI decisions.

- Personality: supportive coach, premium but approachable, warm dark, not gym-bro neon.
- Use the token vocabulary from `DESIGN.md`: `background`, `surface-1`, `surface-2`, `primary`, `text-primary`, `radius-md`, etc.
- Accent color is sparse. Do not flood cards or backgrounds with `primary`.
- Data is the hero: big, glanceable numbers; clear labels; generous whitespace.
- Design for gym context: one-handed use, large touch targets, dim lighting, 1-2 second glances.
- Avoid generic AI UI: centered icon cards, gradient headers, neon glows, fake dashboard clutter, decorative elements with no job.
- iOS should feel Apple-native. Android should feel Material-native. Information architecture stays aligned across platforms.
